# MoM (Movie Match)

영화 메타데이터와 사용자 활동 기록을 바탕으로 개인화 추천을 제공하는 영화 추천 웹 서비스입니다.

---

## 메모

- 메인화면은 영화 목록에서 추천 카테고리 목록으로 변경했습니다.
- 마이페이지랑 메인화면 변경하면서 어두운 계열 스타일로 한번 해봤습니다. 기존꺼랑 비교해서 밝은 계열이나 어두운 계열 선택하면 좋을 것 같아요.
- 회원가입할 때 온보딩으로 선호장르, 영화, 구독 OTT정보 받는 로직 있으면 좋을 것 같습니다.
- 마이페이지에서 현재 가입한 OTT정보 수정 같은 기능 있으면 좋을 것 같습니다.
- 영화 정보에서 OTT에서 제공하는 영화 URL로 이동해야하는데 이거 어떻게 해야할지 생각해보면 좋을 것 같습니다. 지금은 TMDB에서 제공하는 정보만 받고 있습니다.
- 추가적으로 키노라이츠에서 구현할 수 있는 기능 최대한 구현하면 좋을 것 같습니다.

- 더미 테스트 계정
  - `testuser01` ~ `testuser16`
  - 비밀번호는 모두 동일: `Test1234!`
- 추천 결과가 유저마다 다르게 나오는지 확인하기 위해 만든 계정들입니다.
- 스릴러형, 로맨스형, 가족/애니형, 액션형, 감독 중심형, 배우 중심형, OTT 제약형 등으로 나뉘어 있습니다.

예시 계정
- `testuser01 / Test1234!`
- `testuser02 / Test1234!`
- ...
- `testuser16 / Test1234!`

추가 메모
- 키노라이츠에 있는 기능을 최대한 참고해서 가져온 형태입니다.
- 프로젝트 상황에 맞게 일부 기능은 수정했고, 추후 더 다듬을 예정입니다.
- 버튼에 들어간 문구(예: 기능설명, 추천이유)는 현재 개발/시연 확인용으로 넣어둔 상태입니다.
- 실제 시연이나 운영 단계에서는 제거하거나 축약할 필요가 있습니다.
- 영화별 OTT 제공 사이트 정보도 필요하지만, 아직 어떤 API나 데이터 소스에서 가져올지 확정하지 못한 상태입니다.
- 추후에는 사용자가 가입한 OTT를 직접 관리하고, 보유 OTT 기준으로 추천/필터링할 수 있는 기능도 넣으면 좋을 것으로 보고 있습니다.

## 1. 기능 설명

이 프로젝트는 영화 메타데이터와 사용자 활동 기록을 바탕으로 개인화 추천을 제공하는 영화 추천 웹 서비스입니다.

### 전체 흐름
- **메인 홈**: 전체 영화 목록과 추천 영화 목록 제공
- **영화 상세 페이지**: 영화별 반응 및 상태 기록
- **마이페이지**: 사용자의 영화 활동 기록과 취향 확인

### 영화 상세 페이지 기능

#### 좋아요
- 한 번 누르면 좋아요가 켜집니다.
- 다시 누르면 해제됩니다.
- 별로에요와 동시에 켜질 수 없습니다.

#### 별로에요
- 한 번 누르면 별로에요가 켜집니다.
- 다시 누르면 해제됩니다.
- 좋아요와 상호배타입니다.

#### 나중에 볼래요
- 한 번 누르면 저장됩니다.
- 다시 누르면 해제됩니다.

#### 보는중 / 봤어요
- 3단계 토글 방식입니다.

`OFF -> WATCHING(보는중) -> WATCHED(봤어요) -> OFF`

#### 별점
- 별점은 **봤어요(WATCHED)** 상태일 때만 보입니다.
- 1~5점으로 저장할 수 있습니다.
- 다시 상세 페이지에 들어오면 기존 점수가 유지됩니다.
- 봤어요를 해제하면 별점도 함께 사라집니다.

#### 컬렉션에 추가
- 한 번 누르면 컬렉션에 담깁니다.
- 다시 누르면 제거됩니다.

#### 인생영화
- 인생영화는 **컬렉션에 담긴 영화만** 선택할 수 있습니다.
- 최대 **10개까지** 등록할 수 있습니다.
- 빈 슬롯 10개를 전부 보여주는 방식이 아니라, 현재 개수에 맞춰 필요한 슬롯만 보여주도록 구성했습니다.
- 인생영화 추가는 펼침 방식이 아니라 **모달/오버레이 형태**로 컬렉션 영화를 선택하는 UX를 사용합니다.

### UI / 상호작용
- 버튼 클릭 시 페이지를 새로고침하지 않고 상태가 바로 반영됩니다.
- 그래서 상세 페이지에서 여러 버튼을 눌러도 브라우저 뒤로가기가 덜 꼬이도록 구성했습니다.
- 버튼 hover 시 기능 설명 텍스트가 보이도록 해두었습니다.

### 마이페이지에서 볼 수 있는 항목
- 인생영화
- 좋아요
- 컬렉션
- 나중에 볼래요
- 별로에요
- 보는중
- 봤어요

즉, 단순 추천만 제공하는 것이 아니라 사용자의 영화 취향과 활동 이력을 함께 관리할 수 있는 구조입니다.

## 2. 데이터 처리 설명

이 프로젝트의 데이터 처리는  
**API 원본 적재 -> 정규화 -> 태깅 -> 사용자 활동 연결** 순서로 진행됩니다.

기준 파일
- `schema.sql`
- `StartupDataSyncRunner.java`
- `MovieDataBatchService.java`

### 1) 원본 데이터 적재
영화 원본 데이터는 먼저 raw 테이블에 적재됩니다.

예시
- `kobis_movie_raw`
- `tmdb_movie_raw`

즉, 외부 API에서 받은 데이터를 바로 서비스에 사용하는 것이 아니라 먼저 raw 형태로 저장한 뒤 정제와 정규화 단계를 거칩니다.

### 2) 정규화
그 다음 단계에서는 raw 데이터를 실제 서비스용 구조로 분리합니다.

주요 테이블 구조
- 영화 본체: `movie`
- 장르: `genre`
- 영화-장르 연결: `movie_genre`
- 인물: `person`
- 영화-감독 연결: `movie_director`
- 영화-배우 연결: `movie_actor`
- 키워드: `keyword`
- 영화-키워드 연결: `movie_keyword`
- OTT 제공처: `provider`
- 영화-OTT 연결: `movie_provider`
- 기타 출처 연결: `movie_source`

즉, 영화 1편을 중심으로 장르, 감독, 배우, 키워드, OTT 제공처를 관계형 구조로 분리해 저장하는 방식입니다.

이렇게 나눠두면 추천 계산 시에도
- 어떤 장르를 좋아하는지
- 어떤 배우를 선호하는지
- 어떤 키워드 성향이 강한지
- 어떤 OTT를 자주 보는지

를 각각 독립적으로 계산할 수 있습니다.

### 3) 태그 생성
정규화가 끝나면 메타데이터 기반 태그 생성 단계가 들어갑니다.

관련 파일
- `MovieTagService.java`
- `MovieTagGenerator.java`
- `TagScoreCalculator.java`
- `DefaultRecommendationTagRules.java`

여기서는 아래 정보를 바탕으로 규칙 기반 태그를 자동 생성합니다.
- 제목
- 상영시간
- 개봉연도
- 장르
- 키워드

예시 태그
- `tense`
- `mystery`
- `healing`
- `romantic`
- `with_family`
- `disaster`

중요한 점은, 이 태그를 사람이 수동으로 붙이는 것이 아니라 **메타데이터 기반 점수 계산 규칙으로 자동 생성한다는 점**입니다.

또한 한 영화에 태그를 무한정 붙이지 않고
- 타입별 개수 제한
- 전체 개수 제한

을 두어 노이즈를 줄이고 있습니다.

### 4) 사용자 활동 연결
정규화된 영화 메타데이터와 태그 위에 사용자의 활동 데이터가 연결되면서 추천 계산 기반이 만들어집니다.

정리하면 전체 파이프라인은 다음과 같습니다.

`KOBIS / TMDB 원본 적재 -> 정규화 테이블 분리 -> 영화 태그 자동 생성 -> 사용자 활동 데이터 연결 -> 추천 계산 기반 구성`

## 3. 추천 알고리즘 설명

현재 추천 시스템은 **규칙 기반 태그 중심의 하이브리드 콘텐츠 추천 방식**으로 구성되어 있습니다.

핵심 파일
- `UserPreferenceProfileService.java`
- `RecommendationRankingService.java`
- `RecommendationFeaturePolicy.java`
- `RecommendationBlockService.java`

전체 추천 흐름은 크게 3단계입니다.

1. 사용자 행동을 모아서 취향 프로필 생성
2. 그 프로필을 기반으로 전체 개인화 랭킹 생성
3. 전체 랭킹을 잘라 추천 블록 생성

즉, 추천 블록 자체가 별도 알고리즘이라기보다는 **전체 개인화 랭킹을 기반으로 분류해서 보여주는 구조**입니다.

### 1) 사용자 행동 반영 방식
현재 추천 프로필에 반영되는 핵심 행동은 다음 4가지입니다.

- 인생영화
- 좋아요
- 봤어요
- 나중에 볼래요

가중치는 아래처럼 설정되어 있습니다.

- 인생영화: `5.0`
- 좋아요: `3.0`
- 봤어요: 기본 `2.2`
- 나중에 볼래요: `1.5`

단, 봤어요는 별점으로 보정됩니다.

- 별점 없음: `2.2 * 0.70`
- 1점: `2.2 * 0.15`
- 2점: `2.2 * 0.45`
- 3점: `2.2 * 0.85`
- 4점: `2.2 * 1.05`
- 5점: `2.2 * 1.20`

즉,
- 인생영화는 가장 강한 취향 신호
- 좋아요는 강한 선호 신호
- 봤어요는 별점에 따라 보정되는 시청 신호
- 나중에 볼래요는 약한 관심 신호

로 사용하고 있습니다.

현재 `WATCHING(보는중)`은 강한 추천 신호로 직접 쓰지 않고, `WATCHED(봤어요)` 상태만 추천에 반영합니다.  
반대로 `별로에요`, `컬렉션`은 UI 기록에는 들어가 있지만 추천 점수의 핵심 신호로는 아직 직접 사용하지 않고 있습니다.

### 2) 취향 프로필 생성
추천 프로필은 `user_preference_profile` 테이블에 저장됩니다.

여기에는 유저별로 다음 feature score가 누적됩니다.
- `TAG`
- `CAUTION`
- `GENRE`
- `DIRECTOR`
- `ACTOR`
- `KEYWORD`
- `PROVIDER`

예를 들어 사용자가 좋아요, 인생영화, 봤어요를 남긴 영화들에서
- 어떤 태그가 많이 나왔는지
- 어떤 장르가 많이 나왔는지
- 어떤 감독/배우가 많이 나왔는지
- 어떤 키워드가 반복되는지
- 어떤 OTT 제공처가 겹치는지

를 누적 점수로 저장합니다.

이 과정에서는 노이즈를 줄이기 위한 정책도 들어가 있습니다.

- 배우는 상위 출연진만 사용: `display_order <= 5`
- 키워드는 상위 일부만 사용: `display_order <= 8`
- 범용 키워드는 blacklist 제외
- OTT는 `KR + FLATRATE`만 사용
- broad genre는 감쇠 적용

즉, 단순히 사용자가 본 영화를 그대로 복사하는 것이 아니라,  
그 영화들에서 공통적으로 나타나는 feature를 추출해 취향 프로필로 만드는 구조입니다.

### 3) 전체 개인화 랭킹 생성
전체 추천 결과는 `user_recommendation_result`에 저장됩니다.

추천 후보는 모든 영화를 무작정 다 점수화하는 것이 아니라,  
프로필 feature와 하나라도 겹치는 영화들만 후보로 뽑습니다.

현재 후보에서 제외하는 영화는 다음과 같습니다.
- 인생영화로 넣은 영화
- 좋아요한 영화
- 나중에 볼래요 한 영화
- 이미 봤어요(WATCHED) 처리한 영화

즉, 사용자가 이미 명시적으로 반응한 영화는 추천 상단에 다시 올리지 않도록 구성했습니다.

후보 영화마다 아래 점수를 계산합니다.
- `tag_score`
- `genre_score`
- `people_score`
- `keyword_score`
- `provider_score`
- `penalty_score`

여기서 `people_score`는 감독 + 배우를 합친 점수입니다.

최종 점수는 대략 아래 비중으로 합산됩니다.

- TAG: `0.34`
- GENRE: `0.18`
- PEOPLE: `0.16`
- KEYWORD: `0.08`
- PROVIDER: `0.02`
- POPULARITY: `0.10`
- FRESHNESS: `0.04`
- multi-signal bonus: 최대 `0.05`
- caution penalty: 별도 차감

즉 현재 추천 철학은 다음과 같습니다.
- 가장 중요한 것은 태그
- 그 다음은 장르
- 그 다음은 감독/배우
- 키워드와 OTT는 보조
- 인기도와 최신성은 약한 보정 요소

### 4) 태그 과증폭 방지
예전에는 태그 하나만 맞아도 점수가 쉽게 높아지는 문제가 있었지만,  
현재는 `computeMatch()`에서 정규화를 한 번 더 수행합니다.

단순히 “프로필에서 몇 점 맞았는가”만 보는 것이 아니라
- 프로필 대비 비율
- 몇 개 feature가 맞았는지
- 후보 영화 안에서 해당 feature 비중이 얼마나 되는지

를 함께 계산합니다.

즉,
- 태그 1개만 겨우 맞은 영화
- 태그 여러 개가 실제로 겹치는 영화

를 구분하려는 구조입니다.

### 5) penalty와 다양성
`CAUTION` 태그도 프로필에 저장되고, 랭킹 계산 시 `penalty_score`로 차감됩니다.  
가중치는 아직 크지 않지만 구조는 이미 들어가 있습니다.

또한 최종 랭킹 저장 전에 아주 간단한 diversity re-rank도 한 번 수행합니다.

최근 추천 2개 안에서
- 같은 대표 태그
- 같은 대표 장르
- 같은 대표 감독

이 연속되면 약간 감점해서, 상위 추천이 너무 비슷한 분위기로 몰리지 않도록 했습니다.

복잡한 MMR 방식은 아니고, 유지보수가 쉬운 단순 패널티 방식입니다.

### 6) 추천 이유 생성
추천 결과에는 `reason_summary`도 함께 저장됩니다.

이 값은 영화별로
- 가장 강하게 맞은 태그
- 장르
- 감독 또는 배우
- 부족하면 키워드 / OTT

중에서 1~2개를 골라 문장으로 만듭니다.

예시
- 긴장감 있는 태그 일치
- 미스터리 장르 + 특정 감독
- 감성 태그 + 로맨스 장르

즉, “왜 이 영화가 추천됐는지”를 최소한 설명 가능한 구조로 만들어둔 상태입니다.

### 7) 추천 블록 생성
추천 블록은 별도 추천 엔진이 아니라, 이미 만들어진 전체 랭킹에서 상위 일부만 잘라서 구성합니다.

관련 파일
- `RecommendationBlockService.java`

현재 방식은 다음과 같습니다.
- 전체 랭킹 상위 120개까지를 slice로 사용
- 기본 블록: 회원님을 위한 추천
- 추가 블록 후보:
  - `TAG`
  - `GENRE`
  - `DIRECTOR`
  - `ACTOR`
  - `PROVIDER`
- 블록 최소 개수는 4개
- broad genre는 좀 더 엄격한 샘플 수를 요구

즉, 블록은 “개인화 랭킹 결과를 다시 분류해서 보여주는 뷰 구조”입니다.

### 8) 추천 갱신 방식
현재는 활동이 생길 때마다 즉시 무거운 재계산을 돌리지는 않습니다.  
대신 `user_recommendation_refresh_state`에 `dirty = true`로 표시합니다.

관련 파일
- `RecommendationRefreshStateService.java`

실제 재계산은 주로 다음 두 경우에 수행됩니다.
- 사용자가 홈에 들어왔을 때 `ensureRecommendations()`
- 관리자 배치에서 `refresh-dirty`

즉 구조는 **dirty 표시 후 지연 재계산** 방식입니다.

### 한 줄 요약
현재 추천 시스템은

`영화 메타데이터를 정규화해서 저장하고 -> 그 메타데이터로 규칙 기반 태그를 생성한 뒤 -> 사용자 행동을 가중치로 합쳐 취향 프로필을 만들고 -> 그 프로필과 겹치는 영화를 점수화해서 전체 개인화 랭킹을 만들고 -> 그 랭킹을 다시 잘라 추천 블록으로 보여주는 구조`

입니다.
