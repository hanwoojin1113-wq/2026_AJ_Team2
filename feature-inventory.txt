CineMatch 기능

1. 계정 및 온보딩

- 회원가입/로그인/로그아웃: 기본 계정 생성, 로그인 세션 생성, 로그아웃 기능을 제공한다.
- 온보딩 영화 선택: 회원가입 후 장르별, 영화 후보를 사용자에게 제시하고 사용자는 영화를 선택하면 초기 좋아요/취향 데이터로 반영된다.(cold start 완화)
- 사용자 기본 프로필: 닉네임, loginId, 프로필 이미지, 기본 동물 아바타를 사용해 사용자 정보를 표시한다.
- 동물 아바타 리소스: 기본 프로필 이미지로 사용할 동물 아바타 asset을 제공한다.



2. 공통 화면 구조

- 공통 상단바/좌측바: 상단바에는 알림과 로그인/로그아웃을 두고, 좌측바에는 홈, 랭킹차트, 피드, 리뷰, 친구, 배틀, 프로필 메뉴를 둔다.
- 현재 메뉴 강조: 사용자가 현재 보고 있는 페이지에 맞춰 좌측바 메뉴가 active 처리된다.
- 공통 알림 패널: 상단바 알림 버튼으로 최근 알림을 확인하고, 전체 읽음 처리를 할 수 있다.


3. 홈 및 영화 탐색

- 홈 화면: 메인 배너, 실시간 인기작품, 추천 영화, 인기 영화 섹션을 통해 영화를 탐색할 수 있다.
- 영화 검색/고급 검색: 제목 검색과 장르/태그/키워드 기반 필터링으로 영화를 찾을 수 있다.
- 영화 카드 빠른 액션: 홈과 랭킹의 영화 카드 hover 상태에서 좋아요, 별로예요, 나중에 볼래요, 컬렉션 추가를 빠르게 실행할 수 있다.
- 실시간 인기작품 Top 10: 인기 영화 카드에 제목, 포스터, 장르 박스를 함께 표시한다.


4. 영화 상세 페이지

- 영화 기본 정보 표시: 포스터, 제목, 원제, 제작연도, 러닝타임, 장르, 추천 태그, 감독, 배우 정보를 보여준다.
- 영화 이미지/영상: 영화 이미지 갤러리, lightbox, 예고편/관련 영상 탭을 제공한다.
- 영화 관련 게시물: 해당 영화를 태그한 게시물 그리드를 보여주고, 클릭 시 영화별 게시물 피드의 해당 위치로 이동한다.
- 영화 관람 리뷰: 해당 영화에 등록된 리뷰 목록과 리뷰 작성 영역을 제공한다.
- OTT/극장 보러가기: DB에 저장된 OTT 또는 극장 direct URL이 있으면 provider 로고와 함께 보러가기 버튼을 표시하고, 없으면 없음 상태를 보여준다.
- 영화 평가 기능: 좋아요, 별로예요, 나중에 볼래요, 보는중, 봤어요, 별점, 컬렉션 추가를 지원한다

5. 영화 사용자 평가 및 활동 기록

- 영화 평가 기능: 좋아요, 별로예요, 나중에 볼래요, 보는중, 봤어요, 별점, 컬렉션 추가를 지원한다.
- 평가 상태 즉시 반영: 버튼 클릭 후 페이지 새로고침 없이 사용자 상태가 바로 바뀐다.
- 별점 조건 처리: 봤어요 상태에서 별점을 남길 수 있도록 감상 상태와 별점을 연결한다.
- 인생영화 관리: 마이페이지에서 인생영화를 추가/삭제하고, 공개 프로필에도 인생영화를 보여준다.
- 사용자 활동 기반 추천 반영: 좋아요, 별로예요, 별점, 봤어요, 나중에 볼래요, 인생영화 등의 활동이 추천 알고리즘의 취향 신호로 사용된다.


6. 리뷰 기능

- 영화 리뷰 작성: 봤어요 상태인 영화에 대해 최대 1000자 텍스트 리뷰를 작성할 수 있다.
- 리뷰 1인 1영화 정책: 같은 사용자는 같은 영화에 하나의 리뷰만 유지하며, 다시 작성하면 기존 리뷰가 갱신된다.
- 리뷰 목록/상세: 영화 상세의 리뷰 목록과 별도 리뷰 게시판, 리뷰 상세 페이지를 제공한다.
- 리뷰 좋아요: 다른 사용자의 리뷰에 좋아요를 누를 수 있고, 좋아요 수가 반영된다.
- 리뷰 조회수: 리뷰 상세 페이지 접근 시 조회수가 증가한다.
- 리뷰 알림: 누군가 내 리뷰에 좋아요를 누르면 알림이 생성된다.


7. 게시물 작성 기능

- 영화 태그 게시물 작성: 게시물은 특정 영화를 검색해 태그한 뒤 작성한다.
- 다양한 게시물 유형: 이미지, 동영상, 파일, 투표, 퀴즈, 텍스트형 게시물을 지원한다.
- 다중 이미지 업로드: 이미지 게시물은 최대 5장까지 업로드할 수 있고, 첫 번째 이미지가 대표 이미지가 된다.
- 업로드 UX: drag & drop, 미리보기, 파일 제거, 이미지 용량 경고, 제출 중 로딩 상태를 제공한다.
- 영화 검색 선택 UI: 게시물 작성 중 영화 검색 결과를 보고 태그할 영화를 선택/해제할 수 있다.
- 투표/퀴즈 생성: 질문과 선택지를 입력해 투표형 게시물 또는 정답이 있는 퀴즈형 게시물을 만들 수 있다.


8. 피드 및 게시물 보기

- 전역 피드: /feed에서 전체 사용자의 최신 게시물을 인스타 피드처럼 세로 카드로 보여준다.
- 무한 스크롤: 처음 10개를 보여주고, 아래로 스크롤하면 cursor 방식으로 다음 게시물을 추가 로딩한다.
- 영화별 게시물 피드: /movies/{movieCode}/posts에서 특정 영화에 대한 게시물만 연속으로 볼 수 있다.
- 게시물 상세 페이지: /posts/{postId}에서 단일 게시물을 상세히 볼 수 있다.
- 이미지 캐러셀: 다중 이미지 게시물은 화살표, 도트, 터치 스와이프로 이미지를 넘길 수 있다.
- 게시물 부가 콘텐츠: 동영상 재생, 파일 다운로드, 투표 결과, 퀴즈 정답 확인을 카드 안에서 제공한다.
- 게시물 영화 태그 카드: 게시물에 연결된 영화 정보를 보여주고 영화 상세로 이동할 수 있다.


9. 게시물 상호작용

- 게시물 좋아요: 게시물 좋아요를 토글하고 좋아요 수를 표시한다.
- 게시물 댓글: 게시물에 댓글을 작성하고 댓글 목록을 볼 수 있다.
- 댓글 작성자 정보: 댓글에는 프로필 이미지, 닉네임, 작성일, 내용이 표시되고 프로필로 이동할 수 있다.
- 게시물 작성자 팔로우: 피드/게시물 카드에서 작성자를 바로 팔로우 또는 언팔로우할 수 있다.
- 게시물 알림: 내 게시물에 좋아요나 댓글이 달리면 알림이 생성된다.
- 투표/퀴즈 참여: 피드에서 투표에 참여하거나 퀴즈 정답을 선택하고 결과를 확인할 수 있다.


10. 팔로우 및 친구 기능

- 친구 페이지: /people에서 사용자 검색, 추천 사용자, 팔로우/언팔로우 기능을 제공한다.
- 사용자 검색: 닉네임 또는 loginId 기반으로 다른 사용자를 찾을 수 있다.
- 취향 기반 추천 사용자: 영화 취향이 비슷한 사용자를 추천한다.
- 팔로워/팔로잉 목록: 마이페이지와 공개 프로필에서 팔로워와 팔로잉 관계를 확인할 수 있다.
- 공개 프로필 팔로우: 다른 사용자의 공개 프로필에서 바로 팔로우/언팔로우할 수 있다.
- 피드 내 팔로우: 게시물 카드에서도 작성자를 바로 팔로우할 수 있다.


11. 프로필 및 마이페이지

- 마이페이지 활동 요약: 좋아요, 별로예요, 봤어요, 보는중, 나중에 볼래요, 컬렉션, 인생영화 등을 모아 보여준다.
- 공개 프로필: 다른 사용자의 인생영화, 감상 영화, 좋아요 영화, 대표 뱃지, 팔로우 상태를 확인할 수 있다.
- 프로필 이미지 변경: 사용자가 자신의 프로필 이미지를 업로드해 변경할 수 있다.
- 취향 시각화: 사용자 취향 분포를 차트 또는 취향 맵 형태로 보여준다.
- 대표 뱃지 표시: 사용자가 선택한 대표 뱃지를 프로필과 피드 작성자 영역에 표시한다.


12. 뱃지/업적 기능

- 활동 기반 뱃지: 가입, 첫 감상, 첫 게시물, 첫 팔로우, 영화 활동량, 리뷰 활동, 소셜 활동 등에 따라 뱃지를 부여한다.
- 상위 활동 뱃지: 시네필, 영화 매니아, 커뮤니티 스타처럼 누적 활동이 많은 사용자를 위한 상위 뱃지가 있다.
- 뱃지 진행도: 아직 획득하지 못한 뱃지는 진행도를 보여준다.
- 대표 뱃지 선택: 획득한 뱃지 중 하나를 대표 뱃지로 선택하거나 해제할 수 있다.


13. 알림 기능

- 알림 목록/읽음 처리: 상단바 알림 패널에서 알림을 확인하고 전체 읽음 처리할 수 있다.
- 팔로우 알림: 누군가 나를 팔로우하면 알림을 생성한다.
- 게시물 알림: 내가 팔로우한 사용자의 새 게시물, 내 게시물 좋아요, 내 게시물 댓글 알림을 제공한다.
- 리뷰 알림: 내 리뷰에 좋아요가 눌리면 알림을 제공한다.
- 영화 공유/던지기 알림: 친구가 영화를 공유하거나 던졌을 때 알림을 제공한다.
- 상태 변화 알림: 던진 영화가 보는중/봤어요 상태로 바뀌면 관련 알림을 제공한다.
- 알림 링크 이동: 알림 클릭 시 게시물, 댓글 영역, 영화 상세, 사용자 프로필 등 적절한 위치로 이동한다.


14. 영화 던지기 및 공유

- 영화 공유: 영화 상세에서 팔로우한 사용자에게 영화를 공유할 수 있다.
- 영화 던지기: 상호 팔로우한 친구에게 영화를 추천하듯 던질 수 있다.
- 던질 영화 추천: 친구와의 취향 관계를 바탕으로 던질 만한 영화를 추천한다.
- 보낸/받은 영화 조회: 내가 보낸 영화와 받은 영화를 API로 확인할 수 있다.
- 던진 영화 상태 추적: 상대가 던진 영화를 보는중/봤어요로 바꾸면 알림으로 추적할 수 있다.


15. 영화 배틀

- 영화 배틀 기능: 두 영화가 맞붙는 배틀 콘텐츠를 제공한다.
- 배틀 투표 기능: 사용자는 두 영화 중 하나에 투표할 수 있고, 선택을 변경할 수 있다.
- 배틀 결과 표시: 득표 수와 비율을 막대 형태로 보여준다.
- 배틀 자동 생성: 장르, 감독, 배우, 태그, 시대 기준으로 새로운 영화 배틀을 생성할 수 있다.


16. 랭킹/차트

- 랭킹 메인/상세: 여러 랭킹 카테고리를 한 화면에서 보고, 각 랭킹 상세 페이지로 이동할 수 있다.
- 흥행 랭킹: 역대 매출, 천만 클럽, 세계 흥행, 오래 사랑받은 영화 등 흥행 중심 랭킹을 제공한다.
- 품질/평점 랭킹: 평점 명작, 유저 추천 영화 등 평가 기반 랭킹을 제공한다.
- 장르/인물 랭킹: 장르별 1위, 배우 대표작, 감독 흥행작 등 인물/장르 기준 랭킹을 제공한다.
- 특수 지표 랭킹: 가성비 영화, 반짝 흥행, 특별관 선호, 스크린 독점 없이 흥행 같은 차별화된 랭킹을 제공한다.
- 실시간 인기 랭킹: 현재 인기 있는 영화 흐름을 랭킹으로 보여준다.


17. 추천 알고리즘

- 사용자 취향 프로필 생성: 좋아요, 별로예요, 별점, 봤어요, 나중에 볼래요, 인생영화 활동을 feature vector로 변환한다.
- 영화 태그 기반 추천: 영화에 붙은 장르/키워드/상황/분위기/주의 태그를 추천 점수에 반영한다.
- 긍정/부정 신호 반영: 좋아요와 높은 별점은 긍정 신호, 별로예요와 낮은 별점은 부정 신호로 사용한다.
- 개인화 랭킹 생성: 사용자의 취향 프로필과 후보 영화의 feature를 비교해 추천 순위를 만든다.
- 다양성 보정: 같은 장르, 감독, 태그가 추천 상위권을 과도하게 차지하지 않도록 재정렬한다.
- 추천 블록 구성: 개인화 추천 결과를 여러 화면용 추천 블록으로 나누어 제공한다.
- 협업 추천: 취향이 비슷한 사용자의 인생영화를 활용해 별도 추천 섹션을 만든다.
- 추천 자동 갱신: 사용자 활동이 바뀌면 추천 결과를 dirty 상태로 표시하고 필요 시 재계산한다.


18. 영화 태깅 시스템

- 규칙 기반 자동 태깅: 영화의 장르, 키워드, 러닝타임을 바탕으로 추천 태그를 자동 생성한다.
- 태그 점수 계산: 태그별 positive/negative 조건과 필수 조건을 적용해 0~100 점수를 계산한다.
- 태그 타입 관리: MOOD, CONTEXT, THEME, CAUTION 등 태그 타입별 개수 제한을 적용한다.
- 키워드 부족 영화 보완: TMDB 키워드가 부족한 영화도 장르 조합만으로 일부 테마 태그를 받을 수 있게 보완한다.
- 태그 재구축/통계: 관리자 API로 전체 태그를 다시 만들거나 태그 분포를 확인할 수 있다.


19. 영화 데이터 수집 및 정규화(DB에 적재하는 함수 같은거)

- KOBIS 데이터 수집: 박스오피스와 연도별 대표 영화를 KOBIS API에서 수집한다.
- TMDB 데이터 수집: 인기 영화, 고평점 영화, discover 조건 기반 영화를 TMDB API에서 수집한다.
- KOBIS 정규화: KOBIS raw 데이터를 영화, 장르, 감독, 배우, 제작사 정보로 정규화한다.
- TMDB 정규화: TMDB raw 데이터를 포스터, 줄거리, 평점, 인기도, 키워드, 제공서비스, 이미지, 영상 정보로 보강한다.
- KOBIS/TMDB 매칭: 영화 제목, 원제, 개봉일, alternative title 등을 활용해 KOBIS 영화와 TMDB 영화를 연결한다.
- 중복 방지: movie_source, movie_cd, 제목, 제목+개봉일 기준으로 중복 movie row 생성을 방어한다.
- 트렌딩 데이터 갱신: TMDB trending snapshot을 주기적으로 갱신해 인기 영화 데이터를 유지한다.
- 전체 파이프라인 실행: 관리자 API로 수집, 정규화, 태깅, 추천 관련 작업을 한 번에 실행할 수 있다.


20. 예고편/외부 영상

- YouTube 예고편 검색: 영화별 예고편을 YouTube API로 검색한다.
- 예고편 자동 보강: 예고편이 없는 영화에 대해 비동기 작업으로 영상 정보를 채운다.
- 예고편 상태 확인: 예고편 보강 작업의 진행 상태를 확인할 수 있다.
- 예고편 초기화: 특정 영화의 예고편 정보를 초기화할 수 있다.
- 영화 상세 영상 표시: 영화 상세 페이지에서 예고편과 관련 영상을 보여준다.


21. OTT/극장 링크 수집 및 관리

- Kinolights 기반 링크 수집: 공개 페이지에서 OTT/극장 보러가기 URL을 수집하는 크롤러를 제공한다.
- 크롤링 결과 DB 적재: 수집된 URL을 CSV로 저장한 뒤 관리자 API로 DB에 import한다.
- 크롤링 상태 관리: 성공, 링크 없음, 제목 없음, 실패 상태를 저장해 중복 크롤링을 줄인다.
- provider별 링크 관리: 같은 영화의 Netflix, TVING, Wavve, Watcha, Coupang Play, CGV, 롯데시네마, 메가박스 등을 provider 단위로 저장한다.
- provider 로고 관리: OTT와 극장 provider별 로고 asset을 사용해 영화 상세의 보러가기 버튼을 구성한다.
- OTT API PoC: Streaming Availability API와 Watchmode API의 한국 OTT direct URL 제공 품질을 비교하는 별도 스크립트를 제공한다.


22. 관리자/개발 도구

- 데이터 파이프라인 관리자 API: KOBIS/TMDB import, normalize, tag rebuild, recommendation rebuild를 API로 실행할 수 있다.
- 배치 실행 이력: 파이프라인 실행 기록과 결과를 batch_job_run 형태로 저장한다.
- 추천 검증 API: 특정 사용자의 추천 프로필, 랭킹, 추천 블록 상태를 확인할 수 있다.
- 테스트 데이터 seed: 테스트 사용자, 사용자 활동, 커뮤니티 데이터를 생성하는 관리자 기능이 있다.
- OTT 후보 CSV export/import: 크롤링 후보를 CSV로 뽑고, 크롤링 결과를 CSV로 다시 import할 수 있다.
- H2 file DB 기반 로컬 실행: 로컬 파일 DB에 영화/사용자/소셜 데이터를 유지하며 개발할 수 있다.
- 업로드 파일 정적 제공: 게시물 이미지, 동영상, 첨부파일을 업로드 후 정적 URL로 접근할 수 있다.


23. 보조 스크립트 및 리소스

- Kinolights OTT crawler: 영화별 보러가기 URL을 수집하는 Python 스크립트가 있다.
- OTT API 비교 PoC: Streaming Availability와 Watchmode API 결과를 CSV로 비교하는 Python 스크립트가 있다.
- CSV 결과 관리: 크롤링 후보, 크롤링 결과, 외부 API 비교 결과를 CSV로 관리한다.
- provider 로고 리소스: OTT/극장 로고 이미지를 static asset으로 관리한다.
